### Migration 038: Aislamiento de datos por oposición

```sql
-- 1. Añadir oposicion_id a tests_generados
ALTER TABLE tests_generados ADD COLUMN oposicion_id UUID REFERENCES oposiciones(id);

-- Backfill desde tema_id → temas.oposicion_id
UPDATE tests_generados t
SET oposicion_id = tm.oposicion_id
FROM temas tm
WHERE t.tema_id = tm.id AND t.oposicion_id IS NULL;

-- Para tests sin tema_id (simulacros), backfill desde profiles
UPDATE tests_generados t
SET oposicion_id = p.oposicion_id
FROM profiles p
WHERE t.user_id = p.id AND t.oposicion_id IS NULL;

-- Index para queries filtradas
CREATE INDEX idx_tests_user_oposicion ON tests_generados(user_id, oposicion_id);

-- 2. Añadir oposicion_id a flashcards
ALTER TABLE flashcards ADD COLUMN oposicion_id UUID REFERENCES oposiciones(id);
UPDATE flashcards f SET oposicion_id = tm.oposicion_id
FROM temas tm WHERE f.tema_id = tm.id AND f.oposicion_id IS NULL;
-- Fallback
UPDATE flashcards f SET oposicion_id = p.oposicion_id
FROM profiles p WHERE f.user_id = p.id AND f.oposicion_id IS NULL;
```

## 2. SISTEMA DE SUPUESTOS PRÁCTICOS — Diseño completo

### 2.1 Análisis de supuestos reales del INAP

De los 4+ exámenes analizados, el patrón es siempre:
- **Escenario**: funcionario A2 recién incorporado en un ministerio/organismo
- **5 cuestiones** con 2-4 sub-preguntas cada una
- **SIEMPRE mezcla los 3 bloques**: contratación (LCSP) + presupuestos (LGP) + personal (TREBEP) + al menos 1 de: responsabilidad patrimonial, subvenciones, procedimiento administrativo
- **Temas recurrentes (100% de exámenes)**: contrato menor, documentos contables, situaciones administrativas
- **Temas frecuentes (75%)**: responsabilidad patrimonial, subvenciones, notificaciones electrónicas

**Rúbrica oficial INAP (2024)**:

| Criterio | Puntos | Peso |
|----------|--------|------|
| Capacidad para aplicar conocimientos a situaciones prácticas | 0-30 | 60% |
| Capacidad de análisis | 0-10 | 20% |
| Sistemática (estructura, orden lógico) | 0-5 | 10% |
| Capacidad de expresión escrita | 0-5 | 10% |
| **TOTAL** | **0-50** | **100%** |

### 2.2 Flujo del usuario

```
1. /supuesto-practico → elige modo: Guiado (con pistas) o Examen Real (150 min, sin ayuda)
2. La IA genera caso realista con 5 cuestiones (basado en patrones reales INAP)
3. El usuario escribe respuestas en editor de texto (1 campo por cuestión)
4. "Enviar para corrección" → consume 1 crédito de supuesto
5. La IA corrige con la rúbrica INAP + legislación como contexto
6. Resultado: puntuación /50, feedback por cuestión, artículos que faltaron, respuesta modelo
7. Historial para ver progreso
```

### 2.3 Prompts de generación y corrección

**Generación** (system prompt):
- Generar caso realista de funcionario A2 en ministerio/OA
- Obligar mezcla de Bloques IV+V+VI
- Incluir siempre: 1 contrato, 1 cuestión presupuestaria, 1 de personal
- Output: JSON con contexto + 5 cuestiones + sub-preguntas + leyes relevantes

**Corrección** (system prompt):
- Actuar como tribunal INAP con la rúbrica oficial
- Recibir: caso + respuestas del usuario + legislación completa relevante
- Evaluar 4 dimensiones de la rúbrica
- Output: puntuación /50 + feedback por cuestión + artículos faltantes + respuesta modelo
