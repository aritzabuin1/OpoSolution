# ADR: Scope MVP — 3 Leyes, Sin Oral/Audio

**ID**: ADR-0014
**Fecha**: 2026-02-16
**Estado**: ✅ Aceptado
**Autor**: Claude / Aritz

## 1. Contexto y Problema

El PLAN.md original define 5 fases (22 semanas) con features como simulador oral, audio-learning, plan adaptativo, IPR, rankings, alertas BOE email, y 7+ leyes. Con 0 líneas de código escritas y 0 validación de mercado, implementar todo antes de lanzar es un riesgo de "building in the dark" — construir features que nadie quiere.

## 2. Opciones Consideradas

- **Opción A**: Implementar todo (22 semanas) — Máximo producto pero máximo riesgo de desperdicio si el mercado no responde.
- **Opción B**: MVP con 3 leyes core + tests + corrector + pricing (10-14 semanas) — Validar con lo mínimo que demuestra valor diferencial (verificación determinista).
- **Opción C**: MVP ultra-mínimo solo tests (6-8 semanas) — Demasiado básico para diferenciarse de OpositaTest.

## 3. Decisión Elegida: MVP con 3 leyes core (Opción B)

### Scope del MVP

**Incluido:**
- 1 oposición: Auxiliar Administrativo del Estado
- 3 leyes core: CE (Constitución), LPAC (Ley 39/2015), LRJSP (Ley 40/2015)
- Generación de tests con verificación determinista
- Corrector de desarrollos
- Free tier (5 tests + 2 correcciones + blur en tests 4-5)
- Compra tema (4.99€) + Pack (29.99€) + Premium (12.99€/mes)
- Dashboard básico con progreso
- Rachas + logros básicos (5 logros)
- Flashcards simples (auto-generadas, intervalos fijos)
- PWA instalable
- Landing page con SEO

**Excluido del MVP (post-validación):**
- Simulador oral (Whisper + TTS + grabación = 3 integraciones complejas)
- Audio-learning / TTS (coste ElevenLabs alto, sin validar demanda)
- Plan de estudio adaptativo (nice-to-have, no core)
- IPR (requiere >500 usuarios para comparación significativa)
- Rankings/leaderboards (requiere >100 usuarios/oposición)
- Alertas BOE email (monitor BOE manual en MVP, alertas en v2)
- Segunda oposición (validar primero con una)
- Gamificación avanzada (badges extendidos, rankings)
- Verificación v2 (normalización semántica con Claude)
- OAuth Google (email + magic link suficiente)
- Leyes P1/P2: TREBEP, LTBG, LCSP, LOPDGDD, LGP

### Justificación de las 3 leyes

| Ley | Cobertura temario | Razón |
|-----|------------------|-------|
| CE (Constitución) | Temas 1-5 (~20%) | Base de todo opositor. Siempre cae. |
| LPAC (Ley 39/2015) | Temas 6-12 (~28%) | Procedimiento administrativo = core de Auxiliar |
| LRJSP (Ley 40/2015) | Temas 13-18 (~24%) | Régimen jurídico complementa LPAC |

**Total: ~72% del temario cubierto con solo 3 leyes.** Suficiente para validar el producto.

## 4. Consecuencias

- **Positivas**: Time-to-market de 10-14 semanas (vs 22). Valida hipótesis core (¿los opositores pagan por tests verificados con IA?) antes de invertir en features secundarias. Reduce riesgo económico.
- **Negativas**: El 28% del temario no está cubierto en MVP. Opositores que necesitan TREBEP o LTBG no tendrán contenido. Riesgo de parecer "incompleto" frente a OpositaTest (que cubre todo el temario con preguntas genéricas).
- **Mitigación**: Comunicar transparentemente: "Estamos en beta con CE, LPAC y LRJSP. Más leyes próximamente." El diferenciador (verificación determinista) compensa la menor cobertura. Añadir leyes P1 en sprints de 1-2 semanas cada una post-lanzamiento.
